# Session Summary — March 20, 2026

## Features Implemented

### 1. Admin/User Dashboard Separation

- Admin users visiting `/dashboard` are now redirected to `/admin` via middleware
- Admin nav has "Dashboard" as first item pointing to `/admin`
- Removed redundant standalone "Admin" link from nav
- ExamForge logo links to `/admin` for admins, `/dashboard` for students

### 2. Admin Nav Overflow Fix

- Compact layout with `gap-1` and pill-style items for admin nav (12 items)
- Items grouped into "core" (Dashboard, Questions, Generate, Exam) and "content" (Scraper, Discovery, Ingest, Syllabus, Tutorials, Learn, Find, Saved) with visual separators
- Shorter labels: "Questions" instead of "Question Bank", "Find" instead of "Find Content"
- Scrollable horizontal nav with `overflow-x-auto`

### 3. Role-Aware User Menu

- Admin users see purple "Admin" badge instead of plan badge
- Admin profile link goes to `/admin/settings` (not `/dashboard/profile`)
- No "Unverified" email badge for admins
- No subscription query fired for admin users

### 4. Verification Banner Disabled for Admin

- `{!isAdmin && <VerificationBanner />}` — admins never see email/OTP verification prompts

### 5. Enhanced Exam Selector (Generate Questions — Admin)

- Replaced basic dropdown with searchable combobox from portal examination entries
- Shows exam name, category number, exam date, department, syllabus availability badge
- Portal examination entries loaded via new `portalIngestion.listAllExaminations` endpoint

### 6. 4 AI Providers for Question Generation

- Added GPT-4o (OpenAI) and Gemini 2.0 Flash (Google) alongside Claude and Mistral
- Added Perplexity as 5th provider option
- Color-coded provider cards in the provider selector
- Proper error handling for quota exceeded (429) and auth (401) errors

### 7. Syllabus-Aware Question Generation

- When exam has syllabus, topic content is fetched as primary context for the LLM prompt
- Existing questions for the topic are fetched and passed to avoid duplicates
- `examId` made optional — generation works with exam name when no ID is mapped
- `syllabusId`, `syllabusName`, `syllabusNodeId`, `topicName` saved with generated questions
- Subject field made optional

### 8. Generation Progress Settings Display

- Shows "Generation Settings" panel during question streaming: exam, provider (color-coded badge), syllabus, topic, subject, difficulty, question type

### 9. User Examination Selection (Dashboard & Learn)

- Dashboard "Your Examinations" section always visible with "Add Examination" button
- Empty state with icon and "Browse Examinations" CTA when no exams selected
- Learn page header includes "Add Examination" button
- BrowseExamsDialog with search, plan limits (Free: 3, Paid: per plan), upgrade prompt
- Start Exam page pre-populated with user's opted exams via `exam.listForUser`

### 10. General AI Chat Agent

- Full-page chat at `/dashboard/ai-chat` with:
  - Collapsible conversation sidebar with search
  - 5 provider selector (Claude, GPT-4o, Gemini, Mistral, Perplexity)
  - Markdown rendering with `[[suggest:]]` clickable prompt chips
  - Welcome screen with 4 starter prompts
  - Optimistic UI for user messages
  - URL-based conversation deep linking (`?conversation=uuid`)
- Chat history page at `/dashboard/ai-chat/history` with:
  - Usage stats cards (conversations, tokens in/out, cost)
  - Per-provider breakdown
  - Paginated conversation list with delete confirmation
- "AI Chat" added to student navigation

## Database Changes

### New Table: `ai_conversations`

| Column                | Type              | Description                           |
| --------------------- | ----------------- | ------------------------------------- |
| `id`                  | UUID (PK)         | Auto-generated                        |
| `user_id`             | UUID (FK → users) | Conversation owner                    |
| `title`               | varchar(500)      | Auto-generated from first message     |
| `messages`            | JSONB             | Array of `{role, content, timestamp}` |
| `message_count`       | integer           | Number of messages                    |
| `ai_provider`         | varchar(50)       | Last used provider                    |
| `total_input_tokens`  | integer           | Cumulative input tokens               |
| `total_output_tokens` | integer           | Cumulative output tokens              |
| `total_tokens`        | integer           | Cumulative total tokens               |
| `estimated_cost_usd`  | real              | Cumulative estimated cost             |
| `keyword`             | varchar(200)      | Optional topic tag                    |
| `created_at`          | timestamp         | Creation time                         |
| `updated_at`          | timestamp         | Last update time                      |

**Indexes:** `idx_ai_conv_user` (userId), `idx_ai_conv_user_updated` (userId, updatedAt)

**Migration required:** Run `pnpm db:generate && pnpm db:migrate`

### Modified Validators (affecting API contracts)

- `generateQuestionsInputSchema`: `examId` now optional, added `examName`, `syllabusId`, `syllabusName`, `syllabusNodeId`, `topicName`; `subject` now optional
